# 点赞收藏管理功能优化说明

## 概述

本次优化主要解决了点赞收藏功能在不同页面不能同步的问题，并大幅提升了系统管理员的管理能力。

## 主要优化内容

### 1. 统一状态管理

#### 问题
- 每个组件独立管理点赞收藏状态
- 不同页面间状态无法同步
- 重复的API调用造成性能浪费

#### 解决方案
- 创建了统一的 `useLikeFavoriteStore` 状态管理
- 实现了全局状态同步机制
- 优化了API调用，避免重复请求

#### 核心文件
- `src/store/likeFavorite.ts` - 统一状态管理
- `src/components/LikeFavoriteButton.vue` - 更新后的组件
- `src/components/ResourceInteraction.vue` - 更新后的交互组件

### 2. 系统管理员功能大幅增强

#### 新增功能

##### 统计概览
- **总点赞数** - 系统内所有点赞总数
- **总收藏数** - 系统内所有收藏总数  
- **活跃用户数** - 有交互行为的用户数量
- **被交互资源数** - 被点赞或收藏的资源数量

##### 用户统计管理
- **搜索功能** - 按用户名搜索
- **排序功能** - 按点赞数、收藏数、总交互数排序
- **用户详情** - 查看用户详细统计信息
- **重置统计** - 重置指定用户的点赞收藏统计

##### 点赞记录管理
- **搜索筛选** - 按用户ID、资源ID、用户名搜索
- **日期筛选** - 按时间范围筛选记录
- **批量操作** - 支持批量删除点赞记录
- **单条删除** - 删除指定点赞记录
- **详情查看** - 查看点赞记录详细信息

##### 收藏记录管理
- **搜索筛选** - 按用户ID、资源ID、用户名搜索
- **日期筛选** - 按时间范围筛选记录
- **批量操作** - 支持批量删除收藏记录
- **单条删除** - 删除指定收藏记录
- **详情查看** - 查看收藏记录详细信息

##### 资源统计管理
- **资源排行** - 按点赞数、收藏数、总交互数排序
- **搜索功能** - 按资源名称搜索
- **详情查看** - 查看资源详细统计信息

### 3. 新增API接口

#### 管理员专用接口
```typescript
// 删除指定点赞记录
adminDeleteLike(likeId: number)

// 批量删除点赞记录
adminBatchDeleteLikes(likeIds: number[])

// 删除指定收藏记录
adminDeleteFavorite(favoriteId: number)

// 批量删除收藏记录
adminBatchDeleteFavorites(favoriteIds: number[])

// 重置用户点赞收藏统计
adminResetUserStats(userId: number)

// 获取资源点赞收藏统计
getResourceLikeFavoriteStats()

// 获取指定资源的详细统计
getResourceDetailStats(resourceId: number)

// 获取指定用户的详细统计
getUserDetailStats(userId: number)

// 导出点赞收藏数据
exportLikeFavoriteData(format: 'excel' | 'csv')

// 获取系统概览数据
getSystemOverview()
```

## 使用指南

### 对于普通用户

1. **点赞收藏操作**
   - 在任何页面进行点赞收藏操作
   - 状态会自动同步到其他页面
   - 无需刷新页面即可看到最新状态

2. **查看个人统计**
   - 访问"我的点赞收藏"页面
   - 查看个人点赞收藏统计
   - 管理个人点赞收藏记录

### 对于系统管理员

1. **访问管理页面**
   - 登录管理员账号
   - 访问"点赞收藏管理"页面

2. **查看系统概览**
   - 查看总体统计数据
   - 监控系统活跃度

3. **用户管理**
   - 在"用户统计"标签页查看所有用户
   - 使用搜索和排序功能快速定位用户
   - 查看用户详细统计信息
   - 必要时重置用户统计

4. **记录管理**
   - 在"点赞记录"和"收藏记录"标签页管理记录
   - 使用搜索和日期筛选功能
   - 支持单条删除和批量删除
   - 查看记录详细信息

5. **资源管理**
   - 在"资源统计"标签页查看资源排行
   - 分析热门资源
   - 查看资源详细统计

## 注意事项

1. **权限控制**
   - 只有系统管理员可以访问管理功能
   - 普通用户只能查看和管理自己的数据

2. **数据安全**
   - 删除操作需要确认
   - 批量操作有数量限制
   - 重要操作有日志记录

3. **性能考虑**
   - 大量数据时使用分页
   - 搜索功能有防抖处理
   - 避免频繁的API调用
